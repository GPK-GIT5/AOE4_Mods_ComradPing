# Game Data (Excluded)

The `game_data/` tree in the source workspace is not published in this repository because it consists entirely of:

- **Extracted Relic / Microsoft game files** (`cardinal.ucs`, SCAR documentation API dumps, scenario SCAR sources, EBP blueprint XML).
- **AoE4World community cache** of per-civilization JSON files (https://data.aoe4world.com — v0.0.2, marked `"__note__": "This file is autogenerated"`).
- **Generated derivatives** built from the two sources above (canonical units/buildings/upgrades/tech catalogues).

None of the inputs above are owned by this project and none of them are redistributable through this repo.

## Original sources

| Source | URL | Notes |
|---|---|---|
| Official AoE IV game data | Extracted locally from your installed copy of the game | Microsoft / World's Edge / Relic |
| AoE4World data dumps | <https://data.aoe4world.com/> | Community service; see their terms for re-use |
| SCAR documentation | Bundled with the AoE IV Content Editor | Microsoft / World's Edge / Relic |

## Subtree shape (for orientation only)

```
game_data/
├── extracted/
│   └── aoe4/
│       ├── cardinal.ucs
│       ├── data/                  # AoE4World per-civ JSON
│       ├── scardocs/              # Essence_*.api
│       ├── scar_dump/             # Relic SCAR sources
│       └── mods_meta_data.lua
├── blueprints/                    # EBP XML extracted from the game
└── generated/
    ├── canonical/                 # canonical-{units,buildings,...}.json
    ├── derived/                   # weapon/armor catalogues, etc.
    └── schema/                    # JSON Schema for the canonical files
```

## Integration points in this repository

The published code in [Gamemodes/Onslaught](../../Gamemodes/Onslaught/) and the published scenarios in [Scenarios](../../Scenarios/) reference blueprint names (EBP IDs), upgrade IDs, and SCAR engine functions whose canonical definitions live in the excluded `game_data/` tree. To work on those references you need a local extraction of the game data.

## Local regeneration

1. Install Age of Empires IV and its Content Editor.
2. Use the editor to extract the game data you need into a local `game_data/extracted/` folder.
3. (Optional) Sync the AoE4World caches into `game_data/extracted/aoe4/data/`.
4. Run the private generation pipeline to produce `game_data/generated/canonical/`.

The pipeline scripts are kept in the internal repository and are not published here.
