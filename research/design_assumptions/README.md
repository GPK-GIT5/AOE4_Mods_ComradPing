# Design Assumptions

Documented assumptions that the design and tooling rely on. Update when an assumption is invalidated or refined.

## What belongs here

- Engine-behavior assumptions that affect SCAR patterns.
- Workspace-structure assumptions that affect documentation and tooling.
- Data-availability assumptions used by workflow guides.

## Suggested assumption template

Use this structure when adding a new assumptions note in this folder:

1. Assumption statement
2. Why it matters
3. Evidence/source
4. Validation method
5. Invalidation trigger and owner

## Current baseline assumptions

- Public repository content remains docs plus SCAR source only.
- Folder-level READMEs are the primary navigation index.
- Private workspace automation/skills are documented but not published.
- Scenario and gamemode runtime trees are navigated through docs mirrors.
